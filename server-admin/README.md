# HeyClaw 专家套件与技能云端管理后台部署指南

本管理控制台运行于宝塔服务器（Baota Server），旨在提供 Kits 和 Skills 配置数据的可视化编辑、套件 ZIP 包上传以及带双重安全锁的一键备份回退功能。

## 目录结构说明

```text
server-admin/
├── dist/                       # 前端编译后的静态网页资源（已在本地 npm run build 编译好）
├── server.js                   # 后端原生 Node.js 单文件服务
├── backups/                    # 自动生成的备份目录（自动创建）
└── zip-bundles/                # 上传的套件包存放目录（自动创建）
```

---

## 宝塔服务器部署步骤

由于后端 `server.js` 采用 **Node.js 原生 API 编写，无任何第三方 npm 依赖**，部署极为简单，零环境配置。

### 第一步：同步代码至宝塔
将以下两个本地打包产物直接上传至您服务器上的网站目录下（例如 `/www/wwwroot/heyclaw-admin/`）：
1. `server.js` 文件。
2. `dist/` 整个文件夹。

### 第二步：在宝塔中运行 Node 服务
您可以使用以下两种方式之一让服务常驻运行：

#### 方式 A：使用宝塔 Node.js 项目管理器（推荐，可视化操作）
1. 打开宝塔面板，找到并安装 **Node.js项目管理器**。
2. 点击 **添加项目**：
   * **项目目录**：选择刚才上传的 `/www/wwwroot/heyclaw-admin/`。
   * **启动文件**：选择 `server.js`。
   * **项目名称**：输入 `heyclaw-admin`。
   * **运行端口**：输入 `8082`。
   * **环境变量**：
     * 添加 `ADMIN_PASSWORD` = `[您自定义的安全管理员登录密码]`
     * （可选）添加 `DATA_DIR` = `[您当前服务器存放 kit-store.json 的绝对路径]`。如果不指定，默认会在上一级目录 `../server-assets/` 下自动寻找。
3. 点击 **确定**，项目管理器会自动帮您常驻运行该后台，并支持重启和查看日志。

#### 方式 B：使用 PM2 或 nohup 命令行启动
进入对应的目录，在命令行运行：
```bash
# 临时测试启动
ADMIN_PASSWORD="您自定义的密码" DATA_DIR="/www/wwwroot/your-server-assets-path" node server.js

# 使用 pm2 守护进程常驻运行
pm2 start server.js --name "heyclaw-admin" --node-args="--max-old-space-size=512"
```

## 防错一键回退机制说明

本系统在数据安全性上提供了双重保险，可完全防止改错配置导致用户客户端报错：
1. **自动备份**：任何时候只要您在后台点击“保存修改”，系统写入前均会自动在 `backups/` 目录下生成一个克隆文件，带有精准到秒的时间戳，例如 `kit-store.2026-06-23_12-00-00.bak.json`。
2. **一键安全恢复**：
   * 在控制台的 **“备份与安全回退”** Tab 页中，会按时间倒序展示出所有的历史备份记录。
   * 点击任意备份行的 **“恢复此配置”**，系统会立刻用该备份文件覆盖当前的 `kit-store.json` 或 `skill-store.json`。
   * **安全锁**：为了防止操作员选错历史备份文件导致覆盖失误，**在执行回退动作之前，系统会强行对当前的最新的版本再做一次紧急备份**。即便回退错了，您也能一键重新回退至恢复前的那个版本，做到了绝对的“后悔药”机制。

## 专家套件 ZIP 上传机制

1. 在添加或编辑专家套件时，表单下方提供了拖拽上传 `.zip` 的交互区域。
2. 上传后的套件 ZIP 压缩包会自动保存在 `zip-bundles/` 目录下。
3. 表单的 `bundleUrl` 会自动联想并填充为 `/zip-bundles/xxx.zip` 格式的路径，方便与您的静态网站域名搭配下载。
